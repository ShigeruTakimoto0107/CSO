# C-S-O (Console-System-Orchestration)

## 1. プロジェクトの目的
C-S-Oは、Windows PowerShellをバックグラウンドプロセスとして起動し、独自のスクリプトファイル（.csl）を用いて対話的な操作を自動化するためのツールです。
特に、TeraTermマクロ（TTL）に近い操作感をPowerShellで実現することを目的としています。

## 2. システム構成
- **PowerShellController.cs**: PowerShellプロセスの起動、標準入出力のリダイレクト、バッファ管理、および特定の文字列が出現するまでの待機ロジックを担当します。
- **Orchestrator.cs**: スクリプトファイルの解析と実行を担当します。
- **Program.cs**: エントリポイント。引数からスクリプトファイルを読み込み、実行を開始します。

## 3. スクリプト仕様 (.csl)
スクリプトは1行1コマンド形式で記述します。`#` または `;` で始まる行はコメントとして無視されます。

### サポートされているコマンド
| コマンド | 引数 | 説明 |
| :--- | :--- | :--- |
| `sendln` | `[string]` | 指定した文字列をPowerShellに送信し、改行します。 |
| `wait` | `[string]` | PowerShellの出力バッファ内に指定した文字列が出現するまで実行を停止します。 |
| `clearbuffer` | (なし) | 現在蓄積されている出力バッファを空にします。 |

## 4. 開発における重要事項
- **エンコーディング**: 日本語環境での文字化けを防ぐため、PowerShellとの通信には `Encoding.Default` (Shift-JIS) を使用します。
- **待機ロジック**: `wait` コマンドは、一致した文字列までのバッファを消費（削除）する仕様です。
- **未知のコマンド**: 定義されていないコマンドが記述された場合、その行をそのままPowerShellコマンドとして実行し、プロンプト(`>`)が戻るまで待機します。
